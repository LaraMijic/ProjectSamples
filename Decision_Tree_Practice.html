B. Explain - Root & Splitting Value¶

The duration predictor is the root of the tree and the splitting value is the y-target variable. Assuming all values on the left leafnode are less than or equal to a call duration of 249.5 seconds, the caller would be classified as not having a CD because the 'no' value (3268) is larger than the yes value (1116). If a call duration was greater than 249.5 seconds, the caller would be classified as having a CD because the 'yes' value (3524) is greater than the 'no' value (1372).

C. Explain - Mistakes¶

The root node has equal distribution across the two classes (50/50 distribution) and, therefore, a gini score of 0.5, which indicates that there would basically be random assignment to each variable. The white color of the box also indicates that there is no certainty which class the instance would belong to if a prediction were to be made. As the 9280 samples are split in 2, there would be 4640 mistakes. There is inreased prediction confidence when the root is split into two leafnodes. The left leafnode has a 1116 false negative values and the right leafnode has 1372 false positive values. Therefore, the equation would look like 4640 - 1116 - 1372 = 2152 less misclassifications. This is almost 50% less misclassifications.

D. Explain - Entropy¶

Entropy would be 1 in the root node because the proportion of both classes is 0.50. This means there is maximum entropy because it is too difficult for the model to predit with any certainity what an instance could be classified since the samples are split 50/50.

E. Explain - Model Performance¶

The model's performance has decreased a bit. Model 3 has lower precision for both the no and yes classes, a lower f1-score for the yes class, lower accuracy, and lower weighted precision & f1-scores than Model 2. This has likely occured because having an extra variable can give the model more context. It lets the model explore more scenarios and possibilities, which allows it to more accurately predict what will happen in a certain situation given certain factors. As mentioned in the lecture, a good balance must be struck between having too many leaf nodes and too few because hyper specific or broad situations can make it difficult to classify new incoming data.

Task 7 - Describe and compare the three models¶

Model 2 has the highest accuracy at 0.79, whereas the other models both have an accuracy of 0.73. However, as mentioned in the lecture, accuracy is not always the best measure of a model because high accuracy does not always indicate good precesion, recall, or other measures of model performance. Precision indicates how many predicted instances actually belong to a class, whereas recall indicates the model's ability to correctly classify instances to a class. Model 2 has the best precison (p=0.91) for the no class because the no class in model 2 has an average of 17% misclassifications between the 2 levels of leaf nodes while Model 3 has an average of 33.5% misclassifications in the no class between the 2 levels of leaf nodes and the first model has 25.5% misclassifications in the no class. However, model 2 has lower recall for the no class (r=0.65). For the no class, model 1 shows r=0.70 and model 3 shows r=0.87. This means that model 2 has a lower ability to correctly classify instances to class no. Thus, model 2 will return fewer results, but will have higher precision for the results that are returned. Whereas, model 3, for example, will return more results, but they may be less precise than in model 2. When looking at the f-score (blend of precision and recall) for models 2 and 3, they seem to have about the same efficacy (both of their f1-scores are 0.76). Model 1's precision (0.75) and recall (0.70) are pretty close in values, which averages out to a slightly lower f1-score of 0.72. Therefore, if the best model were to be chosen based on f-score, I would select model 2 because it has the highest f-score for the yes class. However, no one metric alone can showcase which model is the best. Nevertheless, when comparing precision (weighted and unweighted), recall (weighted and unweighted), f-score (weighted and unweighted), and accuracy, model 2 seems to perform slighly better than the other two models. This is likely because model 2 is more pruned than model 1 with a lower ccp_alpha value of 0.04, thus increasing the number of leaf nodes and the model's specificity. While model 3 has the same number of leaf nodes as model 2, model 3 does not take duration into consideration, which can reduce its precision because it has less background information to work with.
